## Runtime-Generated Files

These files are created automatically when the app runs. They are gitignored and should **not** be committed:

### `backend/data/cache.db`

SQLite database file. Created on first run.

**Contents (two types of records):**

1. **Query-to-title mappings:**
   ```
   key: "search_query::black hole"
   val: "Black hole"
   ```

2. **RAG answer cache:**
   ```
   key: "black hole::what happens inside a black hole"
   val: {"answer": "...", "sources": [...], "total_chunks": 38, ...}
   ```

### `backend/data/articles/*.json`

One JSON file per fetched Wikipedia article. Filename is the slugified article title.

**Example `virat_kohli.json`:**
```json
{
  "title": "Virat Kohli",
  "summary": "Virat Kohli (born 5 November 1988)...",
  "full_content": "Virat Kohli (born 5 November 1988) is an Indian...\n\nEarly life\n...",
  "url": "https://en.wikipedia.org/wiki/Virat_Kohli",
  "image": "https://upload.wikimedia.org/.../Virat_Kohli.jpg"
}
```

### `backend/data/faiss/<title>.index`

Binary FAISS index file. Contains all embedding vectors for an article. Loaded by `faiss.read_index()`.

**Why binary?** FAISS stores float32 vectors in its own binary format for fastest possible loading. A 50-chunk article index is typically 50 × 384 × 4 bytes = ~75 KB.

### `backend/data/faiss/<title>.chunks.json`

JSON array of the original text chunks, parallel to the `.index` file.

```json
[
  "Virat Kohli (born 5 November 1988) is an Indian international cricketer...",
  "Kohli made his One Day International debut on 18 August 2008...",
  "In Test cricket, Kohli holds the record for the most centuries...",
  ...
]
```

When FAISS returns index positions `[0, 2]`, the app looks up `chunks[0]` and `chunks[2]` to get the actual text.

## Module Import Structure

Python modules use absolute imports prefixed with `app.`:

```python
# In main.py
from app.wikipedia_api import search_wikipedia
from app.rag import run_rag

# In rag.py
from app.embeddings import get_embedding, get_embeddings
from app.vector_store import build_index, search, index_exists, load_index, get_total_chunks
from app.llm import ask_llm
from app.cache import cache
from app.utils import normalize_cache_key

# In wikipedia_api.py
from app.article_store import get_article, save_article
from app.cache import cache
```

This import style works when the server is started with:
```bash
uvicorn backend.app.main:app --reload --port 8001   # from project root
```
or:
```bash
uvicorn app.main:app --reload --port 8001   # from backend/ directory
```

## Naming Conventions

| Item | Convention | Example |
|---|---|---|
| Python modules | `snake_case` | `wikipedia_api.py`, `vector_store.py` |
| Python functions | `snake_case` | `run_rag()`, `search_wikipedia()` |
| Python classes | `PascalCase` | `PersistentCache`, `SearchRequest` |
| React components | `PascalCase` | `App` |
| CSS classes | `kebab-case` | `result-card`, `ai-section` |
| CSS variables | `--kebab-case` | `--bg-card`, `--accent-blue` |
| JSON files | `snake_case` | `virat_kohli.json` |
| FAISS files | Slugified title | `virat_kohli.index` |
| Environment variables | `UPPER_SNAKE_CASE` | `GROQ_API_KEY` |
